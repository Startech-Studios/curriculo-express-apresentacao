Conceito

Orquestração Multi-Agêntica Inteligente

Um novo modelo de operação de marketing digital onde agentes de IA especializados trabalham de forma coordenada, cada um executando com profundidade máxima em sua área, guiados por um orquestrador central que toma todas as decisões estratégicas.

Diferente de um único modelo de linguagem respondendo a tudo de forma genérica, a Orquestração Multi-Agêntica Inteligente funciona como um time de especialistas reais: cada agente carrega um conjunto específico de conhecimento, protocolos e formas de pensar — ativados sob demanda pelo orquestrador. O resultado é precisão especializada em cada entrega, sem perder a visão unificada do todo.

🎯
Especialização Real

Cada agente tem identidade, protocolo e expertise própria. O Mk Franklin não pensa como o Renanzinho — e isso é intencional. Especialização profunda em cada domínio.

🔄
Roteamento Inteligente

James IA analisa cada pedido e decide qual agente aciona, com qual briefing e qual contexto. Nenhum agente age sem direcionamento estratégico prévio.

📦
Base de Conhecimento Central

Todos os agentes consomem os mesmos arquivos de conhecimento do projeto — avatar, produto, linguagem, briefing. Consistência garantida em todas as entregas.

⚡
Execução Paralela

Enquanto James define a estratégia, os agentes podem ser acionados em sequência ou em paralelo — copy, tráfego, marca e produto avançando ao mesmo tempo sem conflito.

🧠
Memória Persistente

O sistema mantém memória do projeto entre conversas. Contexto, decisões e aprendizados acumulados — o squad cresce junto com o projeto ao longo do tempo.

Orquestrador-Chefe

James IA V3.0 — Engenharia & CoreBrain

James não é um agente de execução. É o cérebro estratégico do squad. Antes de qualquer entrega, James pesquisa, analisa e define — garantindo que cada agente execute com o direcionamento certo.

James IA V3.0 combina 20 anos de expertise em copywriting e estratégia de marketing com capacidade de pesquisa profunda, análise de mercado e orquestração de squads. É a fusão de um CMO estratégico com um copywriter sênior — o único agente que pensa no funil completo antes de delegar qualquer execução.

CoreBrain 01
Especialização de Nicho

Lê o briefing do projeto e ativa conhecimento específico do mercado. Para o Currículo Express: empregabilidade, ATS, candidatos em transição, linguagem de carreira. Cada projeto recebe terminologia e gatilhos do seu nicho.

CoreBrain 02
Níveis de Consciência

Mapeia onde o público está na escala de Eugene Schwartz (1–5) antes de definir o ângulo de qualquer peça. Inconsciente do problema até totalmente consciente do produto — cada nível tem copy diferente.

CoreBrain 03
Pensamento de Funil

Antes de qualquer execução, mapeia: de onde vem o lead, o que sabe, o que sente, qual o único objetivo da peça e para onde vai depois. Cada material é uma engrenagem dentro de uma máquina maior.

CoreBrain 04
Pesquisa de Mercado

Conduz pesquisas estruturadas: tamanho de mercado, concorrência direta e indireta, benchmarks de nicho, tendências e linguagem real do público. Toda afirmação precisa de fonte — nunca achismo.

CoreBrain 05
Análise Estratégica

Aplica SWOT com dados reais, define USP validada, Big Idea de campanha e projeta 3 cenários (otimista, realista, conservador) antes de recomendar qualquer estratégia de funil.

CoreBrain 06
Roteamento de Agentes

Identifica qual agente acionar para cada demanda, prepara o briefing estratégico e instrui o usuário com o comando correto. Garante que nenhum agente execute sem direcionamento claro.

Funcionalidades Gerais

📚
Base de Conhecimento

Consulta obrigatória dos 5 arquivos da pasta conhecimento/ antes de qualquer criação: briefing, avatar, produto, linguagem e scripts de referência. Sem esses dados, não escreve.

✍️
Copywriting Direto

Além de orquestrar, James escreve copy. Domina todos os frameworks (AIDA, PAS, PASTOR, BAB, 4Ps) e todos os formatos — emails, landing pages, VSL, anúncios, WhatsApp, scripts de vídeo.

🚫
Protocolo Anti-IA

Lista de palavras e padrões proibidos (imagine, incrível, revolucionário, travessões excessivos, frases binárias). Cada entrega soa como um humano real — não como output automático de IA.

🔁
Versionamento

Gerencia versões A/B de copies, preserva originais, cria iterações com sufixo de versão e documenta as diferenças estratégicas entre cada variação para facilitar os testes.

Os 4 Agentes do Squad

Cada agente carrega uma identidade distinta, protocolo próprio e expertise profunda. Executores de alto desempenho que operam sob direcionamento do James IA V3.0.

✍️
Mk Franklin
"O copy que fecha"
/copywriter

Identidade

O nome não é coincidência. Benjamin Franklin não escrevia para impressionar — escrevia para persuadir, mover, fazer acontecer. Mk Franklin é assim: cada palavra tem propósito, cada frase puxa a próxima, cada parágrafo está a serviço de uma única coisa — a ação do leitor. Escola americana, resultado em português.

Protocolo

Nunca escreve sem os 3 documentos obrigatórios: briefing da campanha, manual de comunicação do expert e estudo de avatar. Se não tem esses dados, pede antes de digitar uma linha. Aplica mapa de contexto interno antes de cada peça: de onde vem o lead, o que sabe, o que sente, qual o único objetivo da peça e para onde vai depois.

Funções e Entregas

  • E-mails: sequências de lançamento, nutrição, perpétuo e recuperação — com assunto, preview, corpo e CTA
  • Páginas de vendas: long-form e short-form com Hero → Problema → Solução → Prova → Oferta → Garantia → CTA → FAQ
  • VSL e scripts de vídeo: Ideia + Roteirização + Observações de gravação + Teleprompter + Bulletpoints
  • Anúncios em imagem: Headline (7 palavras) + Subheadline + Informação + CTA "APERTE EM SAIBA MAIS"
  • Anúncios em vídeo: Hook (3s) + Roteiro + Copy completa + Legenda do anúncio
  • WhatsApp grupos e API: mensagens formatadas nativas, até 900 e 500 caracteres respectivamente
  • Legendas e conteúdo orgânico: Instagram, LinkedIn, TikTok com hook, corpo e CTA de engajamento
  • Planejamento de conteúdo: plano mensal com pilar, ângulo, hook e CTA para cada post
📊
Renanzinho
"O cara dos números que também entende gente"
/gestor-trafego

Identidade

O diminutivo não é à toa — está no miolo da operação, mais perto da campanha do que qualquer outro agente. Analítico, questionador e com raciocínio fora da caixa: não aceita a primeira leitura dos dados, cruza métricas que a maioria não associa. Escala programática é o forte — pensa em sistemas, automações e lógica de bid, não em ajustes pontuais. Por trás de cada CPL, há um humano tomando decisão — e Renanzinho nunca esquece isso.

Protocolo

Coleta briefing completo antes de montar qualquer plano: objetivo, números do negócio, histórico de campanhas, prazo, ativos disponíveis e plataforma. Calcula viabilidade com projeção de CPL, leads, vendas, faturamento e ROAS antes de recomendar qualquer estrutura. Se os números não fecham, diz claramente — nunca valida metas irreais.

Funções e Entregas

  • Planejamento de campanha: estrutura completa com campanhas, adsets, públicos, criativos, orçamentos e nomenclatura
  • Projeção de viabilidade: CPL estimado, leads projetados, ROAS esperado, CPL máximo sustentável
  • Estratégia de criativos: matriz de ângulos (dor, desejo, prova, curiosidade, urgência) com hook e CTA por ângulo
  • Cronograma de execução: pré-lançamento → fase de aprendizado → primeira análise → otimização → escala
  • Plano de escala programática: escala vertical (orçamento), horizontal (duplicação) e por criativos
  • Análise de relatórios: 5 camadas — visão geral, entrega, criativo, público, funil pós-clique
  • Relatório de performance: resumo executivo, painel de métricas, diagnóstico de causa raiz e otimizações priorizadas
  • Configurações técnicas: pixel, eventos, Conversions API, UTMs e naming convention

Dalí Studio
"Visão antes de execução"
/criador-marca

Identidade

Salvador Dalí não criava o que era esperado — criava o que era inesquecível. Dalí Studio opera assim: antes de qualquer traço, há pensamento. Antes de qualquer pixel, há posicionamento. Questiona antes de criar, pesquisa antes de propor. Não cria logos — cria marcas que servem para exatamente uma empresa no mundo.

Protocolo Anti-IA

Antes de entregar qualquer peça, passa pelo filtro: "Se eu mostrasse isso para um designer experiente, ele diria que foi feito por IA?" Se a resposta for sim ou talvez, reprova e refaz. Elimina gradientes genéricos, ícones abstratos, fontes sem personalidade e composições simétricas demais. Justifica cada decisão criativa com base no briefing.

Funções e Entregas

  • Logomarca: conceituação (2-3 conceitos antes de executar) + execução SVG com versões principal, alternativa, favicon e monocromática
  • Branding Book: propósito, logomarca, cores (HEX/RGB/CMYK), tipografia, elementos visuais e aplicações contextualizadas
  • Templates de conteúdo: post feed, carrossel, stories, destaque, banner e thumbnail — em HTML/SVG renderizável
  • Layout de páginas: wireframe → direção visual → HTML/CSS funcional e responsivo (desktop + mobile)
  • Especificações técnicas: paleta completa, escala tipográfica, área de proteção e tamanho mínimo do logo
  • Direcionamento para design: briefing criativo para designers humanos com referências, conceito e diretrizes

Funcionamento

Como Funciona na Prática

Do pedido ao entregável. O fluxo completo de como a Orquestração Multi-Agêntica processa cada demanda e gera output de alta qualidade.

1

Usuário faz um pedido ao James IA V3.0

Toda interação começa com James. O usuário descreve o que precisa em linguagem natural — sem necessidade de especificar qual agente acionar ou qual formato usar.

→ Input: "Preciso de anúncios para o Meta para captar leads frios"
2

James consulta a base de conhecimento

Antes de qualquer ação, James lê os arquivos de conhecimento do projeto: briefing, avatar, produto, linguagem e scripts de referência. Esse passo é obrigatório — garante que tudo que sair do squad está alinhado com o projeto.

→ Lê: conhecimento/01-briefing, 02-avatar, 03-produto, 04-linguagem
3

James analisa e define a estratégia

Identifica o nível de consciência do público, mapeia o contexto da peça no funil, define o ângulo e estrutura o briefing estratégico. Toma a decisão de qual agente acionará.

→ Decisão: "Ângulo DOR · Público frio nível 2 · Acionar Mk Franklin + Renanzinho"
4

James aciona o agente executor com briefing completo

Instrui o usuário a invocar o skill correspondente e fornece todo o contexto necessário para que o agente execute sem lacunas. O agente não precisa perguntar o que James já mapeou.

→ "Acione /copywriter com: [briefing completo de 5 campos]"
→ "Acione /gestor-trafego com: [objetivo, orçamento, público-alvo]"
5

Agente executor entrega com protocolo próprio

Mk Franklin escreve as copies com os 3 documentos, o mapa de contexto e o checklist anti-IA. Renanzinho monta o plano com projeção de viabilidade e estrutura de campanha. Cada agente entrega no seu padrão de excelência.

→ Output Mk Franklin: 3 versões de anúncio (ângulos: dor / desejo / prova social)
→ Output Renanzinho: plano de campanha + projeção de CPL + estrutura de adsets
6

James valida e entrega ao usuário

James revisa a entrega com base nos padrões do projeto, sinaliza se algo precisa de ajuste, sugere próximos passos e registra a decisão na memória do projeto para uso futuro.

→ Entregável final revisado · Salvo na pasta correta · Próximos passos sugeridos

Manual

Como Instalar o Marketing Squad

Guia passo a passo para implementar a Orquestração Multi-Agêntica em qualquer projeto. Pré-requisito: Claude Code instalado e configurado.

📋
Pré-requisitos
  • Claude Code (CLI da Anthropic) instalado — npm install -g @anthropic-ai/claude-code
  • Conta Anthropic com acesso à API
  • Python 3.x instalado (para o DaVinci Pro Max)
  • Git instalado (para versionamento e compartilhamento com a equipe)
1

Criar a estrutura de pastas do projeto

Crie a pasta raiz do projeto e as subpastas necessárias.

# Criar estrutura base mkdir meu-projeto cd meu-projeto mkdir conhecimento .claude/skills Designer mkdir campanhas anuncios emails scripts-videos
2

Copiar o CLAUDE.md (James IA V3.0)

O CLAUDE.md é o prompt mestre do James IA V3.0. Ele vai para a raiz do projeto. É o primeiro arquivo que o Claude Code carrega ao abrir o projeto.

# Copiar o CLAUDE.md do repositório do squad para a raiz cp squad-source/CLAUDE.md ./CLAUDE.md
3

Instalar os 4 agentes como skills

Cada agente é um arquivo SKILL.md dentro de sua respectiva pasta em .claude/skills/. O Claude Code detecta e carrega automaticamente.

# Copiar os 4 agentes cp squad-source/.claude/skills/copywriter/SKILL.md .claude/skills/copywriter/SKILL.md cp squad-source/.claude/skills/gestor-trafego/SKILL.md .claude/skills/gestor-trafego/SKILL.md cp squad-source/.claude/skills/criador-marca/SKILL.md .claude/skills/criador-marca/SKILL.md cp squad-source/.claude/skills/ui-ux-pro-max/SKILL.md .claude/skills/ui-ux-pro-max/SKILL.md
4

Instalar o motor de busca do DaVinci Pro Max

O DaVinci Pro Max usa scripts Python e base de dados CSV para consultas de design. Precisam estar disponíveis na raiz do projeto.

# Copiar scripts e dados do DaVinci Pro Max cp -r squad-source/src/ui-ux-pro-max/scripts ./src/ui-ux-pro-max/scripts cp -r squad-source/src/ui-ux-pro-max/data ./src/ui-ux-pro-max/data # Testar se está funcionando python3 src/ui-ux-pro-max/scripts/search.py "landing page SaaS" --domain landing
5

Preencher a base de conhecimento

Sem a base de conhecimento, James IA não opera. São 5 arquivos obrigatórios na pasta conhecimento/. Você pode preencher manualmente ou pedir ao próprio James IA para ajudar.

# Arquivos obrigatórios conhecimento/01-briefing-projeto.md ← visão geral do projeto conhecimento/02-dossie-publico-alvo.md ← avatar, dores, desejos, objeções conhecimento/03-produto-principal.md ← produto, preço, diferenciais conhecimento/04-manual-linguagem.md ← tom de voz, expressões, proibições conhecimento/05-scripts-referencia.md ← copies aprovadas como referência
6

Abrir o projeto no Claude Code e verificar

Abra a pasta do projeto no terminal com Claude Code. Os skills serão detectados automaticamente e aparecerão disponíveis.

# Abrir Claude Code na pasta do projeto cd meu-projeto claude # Verificar skills carregados (deve aparecer na lista) # /copywriter · /gestor-trafego · /criador-marca · /ui-ux-pro-max
7

Primeira interação — James assume o comando

Ao abrir o projeto, James IA V3.0 assume automaticamente como orquestrador. Comece descrevendo o projeto ou pedindo o que precisa — James lê a base de conhecimento e distribui para os agentes.

# Exemplo de primeira interação "James, preciso de uma campanha completa de captação de leads para o lançamento" # James irá: # 1. Ler os arquivos de conhecimento # 2. Definir a estratégia de funil # 3. Acionar Mk Franklin para copy dos anúncios # 4. Acionar Renanzinho para o plano de mídia
Equipe

Subir no GitHub e Compartilhar com a Equipe

Como versionar o squad, publicar no GitHub e permitir que qualquer membro da equipe clone e comece a operar com os mesmos agentes.

⚠️
O que NÃO vai para o GitHub

Nunca commite informações sensíveis. Antes de subir, verifique se não há chaves de API, credenciais ou dados pessoais de clientes nos arquivos de conhecimento.

1

Criar o .gitignore

Proteja arquivos sensíveis antes de iniciar o repositório.

# .gitignore recomendado .env .env.local *.key node_modules/ __pycache__/ .DS_Store
2

Inicializar o repositório Git

git init git add . git commit -m "feat: Marketing Squad instalado — James IA V3.0 + 4 agentes"
3

Criar repositório no GitHub e conectar

Crie um repositório privado no GitHub (recomendado para projetos de cliente) e conecte ao local.

# Conectar ao repositório remoto git remote add origin https://github.com/sua-org/nome-do-repo.git git branch -M main git push -u origin main
4

Equipe clona e instala

Cada membro da equipe clona o repositório e já tem o squad completo funcionando — com os mesmos agentes, mesma base de conhecimento e mesma estrutura.

# Clonar o squad git clone https://github.com/sua-org/nome-do-repo.git cd nome-do-repo # Verificar Python para o DaVinci Pro Max python3 --version # Abrir o Claude Code claude
5

Fluxo de trabalho em equipe

Cada entregável criado pelo squad vai para a pasta correta e pode ser commitado. A equipe mantém o histórico de todas as copies, campanhas e materiais criados.

# Após criar material com o squad git add anuncios/facebook/anuncio-lead-v1.md git commit -m "feat: anúncios de captação — 3 ângulos — Mk Franklin" git push # Pull para sincronizar com o time git pull
🌿
Branches por Campanha

Crie uma branch para cada lançamento ou campanha. Quando aprovado, merge para main. Mantém o histórico limpo e permite reverter facilmente.

git checkout -b campanha/lancamento-maio
🔄
Atualizar o Squad

Quando houver melhorias nos agentes ou no CLAUDE.md, atualize os arquivos e commite. A equipe recebe as atualizações com um simples git pull.

git pull origin main
